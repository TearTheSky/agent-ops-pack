# AI Agent Operations Package
## AIエージェントと人間の協調作業を支援するパッケージ

### 📖 このパッケージについて

このパッケージは、**AIエージェント（ChatGPT、Claude、Geminiなど）と人間が協調して安全・効率的に運用業務を遂行するためのツール集**です。

**AIの理解が不足している方でも安心して使えるよう、段階的に説明しています。**

## 🤖 AIエージェントって何？

**AIエージェント**とは、ChatGPT、Claude、Geminiなどの**対話型AI**のことです。

### AIエージェントの特徴
- ✅ **24時間働ける**（人間は休憩が必要）
- ✅ **大量の情報を素早く処理できる**
- ✅ **感情に左右されない**
- ✅ **適切な定義書があれば実際の作業を実行できる**
- ❌ **文脈を理解できないことがある**
- ❌ **責任を取れない**（人間が最終責任）

### なぜAIエージェントと協調するの？
- **効率化**: 繰り返し作業を自動化
- **品質向上**: 人間のミスを減らす
- **知識共有**: チーム全体のスキル向上
- **継続的改善**: 作業プロセスの最適化

## 🚀 使い方（初めての方向け）

### Step 1: パッケージをダウンロード
```bash
# このリポジトリをクローンまたはダウンロード
git clone [repository-url]
cd mei-chan-ops-pack
```

### Step 2: AIエージェントの性格を設定
1. `persona/ai_persona_template.yaml` を開く
2. あなたのチームに合うように設定を変更
3. 特に「変更推奨ゾーン」の部分を確認

### Step 3: AIエージェントにペルソナを与え、状況と次のステップを教える

**💡 このステップの重要性**
このステップは、AIエージェントと人間の信頼関係を構築し、効果的な協調作業の基盤を作る重要な段階です。AIエージェントは最初は何も知らない状態なので、時間をかけて段階的に理解を深めていく必要があります。

**AIエージェントも人間と同じように、最初は完璧ではありません。一緒に学びながら改善していきましょう。**



**🤝 成功のコツ**
- **焦らずに**: AIエージェントの理解度に合わせて説明
- **具体的に**: 抽象的な表現は避け、具体的な例を交える
- **段階的に**: 基本情報から詳細情報へと段階的に提供
- **フィードバック**: AIエージェントの質問や提案に積極的に反応

```
「persona/ai_persona_template.yamlの内容を読んで、このペルソナで作業を手伝ってください。
main.yamlを最初に読んで、全体構成を理解してください。

私のプロジェクトについて、作業手順書を作成するために必要な情報を教えてください。
類推せずに、必要な情報はすべて提供を求めてください。
チャットが始まったばかりのAIが無知なのは罪ではありません。なんでも質問してください。

特に以下の情報が必要です：
- 使用するツール（Terraform、GitHub、Slack等）
- 具体的なコマンドやスクリプト
- ファイルパス・作業ディレクトリ
- 権限・認証の方法
- 既存の作業手順書やスクリプトの場所

その後、Step4の作業手順書作成に進むための具体的な提案をしてください。
推奨は「CONFIG.yamlを修正してAIによる自動生成で初期バージョンを作成し、開発環境で試行して反省会で一緒に育てていく」方法です。」
```

### Step 4: 作業手順書を作成（2つの方法）

**💡 このステップの重要性**
作業手順書は、AIエージェントと人間が安全・効率的に協調作業を行うための基盤です。**最初から完璧な手順書を作ることはできません**。AIが作成した初期バージョンを人間がレビューし修正し、開発環境のような安全な場所で試行して、反省会で改善を重ねながら段階的にバージョンアップしていくことが重要です。

**🔄 継続的改善のプロセス**
1. **人間によるCONFIG.yaml修正（サマリ作成）**
2. **AIによる詳細化（operation_guide.yaml作成）**
3. **人間によるレビュー・修正**
4. **開発環境での試行**
5. **反省会での改善点洗い出し**
6. **手順書のバージョンアップ**
7. **本番環境での適用**

#### 方法A: 手動で作成
1. `work_definition/long_operation_guide_template.yaml` をコピー
2. あなたのプロジェクト用にカスタマイズ
3. **AIエージェントが実際に実行できる**具体的な作業手順を記入

#### 方法B: AIに自動生成してもらう（推奨）
1. **CONFIGファイルを修正（人間さんが実施）**
   - `work_definition/CONFIG.yaml` の例を修正してご自身の運用作業にフィットさせる
   - 必要に応じて新規パラメータを別途自由に追加することもできます
   - **CONFIG.yamlは作業のサマリ（概要）として機能**
   

2. **AIエージェントに依頼**
   ```
   「このCONFIGファイルを使って、work_definitionの中に新しいoperation_guide.yamlを作成してください。
   long_operation_guide_template.yamlを参考にして、
   私のプロジェクトに合った具体的な作業手順書を作ってください。
   
   CONFIG.yamlのサマリ（概要）を詳細化して、実行可能な作業手順書にしてください。
   
   特に以下の点を含めてください：
   - 作業開始・完了の記録
   - Slack等のコミュニケーションアプリへの通知の手順
   - 進捗記録の方法
   - エラー時の連絡方法
   - 反省会の実施タイミング
   - 各タスクの具体的な実行手順
   ```

3. **生成されたoperation_guide.yamlを確認・調整**
   - **progress_tracking/progress_tracking_config.yaml** を参照して進捗記録設定を確認・調整
   - **templates/slack_message_templates.yaml** を参照して通知メッセージを調整
   - **retrospective/retrospective_how_to.yaml** を参照して反省会の手順を確認
   - 安全性の確認
   - プロジェクト固有の要件の追加
   - エラーハンドリングの確認
   - 各タスクの実行手順の詳細化
   - **CONFIG.yamlのサマリが適切に詳細化されているか確認**


### Step 5: 実際に使ってみる
- **作業開始**: AIエージェントに作業を依頼
- **進捗確認**: 定期的に状況を確認
- **結果記録**: 作業完了後に進捗を記録
- **チーム連絡**: Slackで結果を報告
- **反省会実施**: **必ず反省会を実施！** どんなに内容が少なくても、短くても、継続的改善のために反省会をやりましょう！
- **次回改善**: 学んだことを次回に活かす

## 💡 よくある質問（FAQ）

### Q: AIエージェントって安全なの？
**A**: AIエージェントは**提案・支援**だけでなく、**適切な定義書・ワークフローがあれば実際の作業も実行**できます。本パッケージは安全な作業手順と実行可能な定義書の共有を目指しています。実際の作業においてはご自身の責任にて適切に加筆・修正を行いつつ利用してください。

### Q: AIの技術的な知識がなくても使える？
**A**: はい！このパッケージは**段階的に説明**されているので、AIの知識がなくても使えます。しかし、いざという時にAIの行動を制御できるようにするために、ご自身が担当するシステム運用作業・開発関連作業については技術的な理解を深める必要があります。

### Q: どのAIエージェントを使えばいい？
**A**: IDEに統合されているAIを利用してください。（Cursor, VS Code, IntelliJ IDEAなど）IDEの中のAIであれば、利用するモデルは、ChatGPT、Claude、Geminiなど、どの対話型AIでも使えます。お好みのものを選んでください。

### Q: カスタマイズは必要？
**A**: 基本的な設定はそのまま使えますが、**あなたのプロジェクトに合わせて調整**することをお勧めします。

### Q: 反省会は本当に必要？
**A**: **絶対に必要です！** どんなに内容が少なくても、短くても、反省会をやりましょう！継続的改善のためには反省会が必須です。

### Q: operation_guideの作成が大変そう...
**A**: **`work_definition/CONFIG.yaml`を修正してAIに依頼する方法**をお勧めします。人間さんはCONFIGファイルの例を修正して業務にフィットさせるだけで、AIが`work_definition/operation_guide.yaml`を自動生成してくれます。各設定ファイルを参照して調整することで、より良い作業手順書が作成できます。

## 📋 各ファイルの役割

### 🤖 `persona/` - AIエージェントの性格設定
**目的**: AIエージェントに「どんな性格で働いてほしいか」を設定
- **変更推奨ゾーン**: 必ずあなたのチームに合わせて変更
- **変更可能ゾーン**: 必要に応じて調整
- **会社文化への配慮**: 言語・コミュニケーションスタイル

### 📝 `work_definition/` - 作業手順書テンプレート
**目的**: AIエージェントが実際に実行できる具体的な作業手順を定義
- **CONFIG.yaml**: 作業の概要とタスク構造を定義
- **long_operation_guide_template.yaml**: 汎用的な作業手順書テンプレート
- **operation_guide.yaml**: 具体的な作業手順書（AIが自動生成）
- **段階的実行**: 安全に作業を進める
- **エラーハンドリング**: 問題が起きた時の対処法
- **時刻計測**: 作業時間の記録
- **事前チェック**: リスクの事前評価
- **実行可能なコマンド**: AIエージェントが実際に実行できる具体的な手順
- **CONFIGベース生成**: 人間さんがCONFIGを書くだけでAIが自動生成
- **連絡・記録**: Slack通知、進捗記録、反省会の手順も含む

### 📊 `progress_tracking/` - 進捗管理システム
**目的**: 作業の進捗と結果を記録
- **スプレッドシート用**: チーム共有用の記録
- **AI作業ログ**: AIエージェント自身の記録
- **エビデンス報告**: 問題追跡用の記録

### 📚 `philosophy/` - 学んだこと・改善点
**目的**: 作業で学んだことを記録・共有
- **シンプルな構造**: 問題・行動・学びの3項目
- **具体的な事例**: 実際に起きた問題と対処法
- **継続的改善**: 次回への活用法

### 🔄 `retrospective/` - 反省会のやり方
**目的**: AIエージェントと人間が効果的に反省会を行う
- **基本原則**: 簡単に理解できる設計
- **5ステッププロセス**: 段階的な反省会
- **具体的な事例**: 実際の反省会例
- **必須実施**: **どんなに内容が少なくても、短くても、必ず反省会を実施！**

### 📄 `templates/` - メッセージテンプレート
**目的**: 統一されたコミュニケーション
- **Slack投稿**: チームへの通知
- **PR説明**: 統一された形式
- **エビデンス報告**: 問題追跡用
